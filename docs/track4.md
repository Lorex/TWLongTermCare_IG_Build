## Track 4

## 賽道 4：長照 2.0 服務計畫與紀錄

### 說明

長照 2.0 服務計畫與紀錄主要針對個案管理師擬定個人化照顧計畫、服務紀錄與追蹤等服務進行標準化資料交換。本賽道包含兩個核心問卷：

1. **AA01 照顧計畫擬訂與服務連結問卷**：用於個案管理師在收到照管中心轉介的長照個案後，前往案家與個案及其家屬討論，根據長照服務額度、問題清單及照顧需求擬定個人化照顧計畫的標準化評估工具。
1. **AA02 照顧管理問卷**：用於個案管理師定期追蹤長照個案的服務使用狀況與需求變化，執行服務品質追蹤，並處理相關的諮詢與申訴的標準化追蹤工具。

參測單位應依照上述需求分別實作相關功能，並提供應用系統以滿足長照 2.0 服務計畫與紀錄的資料交換需求。

### 檢核項目

* 參測單位應實作照顧計畫擬定功能（情境 1：AA01 問卷）
* 參測單位應實作服務紀錄與追蹤功能（情境 2：AA02 問卷）

### 各系統角色於本賽道之作用

| | | |
| :--- | :--- | :--- |
| LTC_MANAGEMENT | LTC Management System | 負責接收來自個案管理師的 AA01 照顧計畫擬定與 AA02 服務追蹤等資料，並提供資料管理與上傳功能。 |
| LTC_CONSUMER | LTC Consumer | 負責向 LTC Repository 查詢 AA01 照顧計畫與 AA02 服務追蹤等資料。 |
| LTC_REPOSITORY | LTC Repository | 負責提供資料交換與儲存功能，包括 AA01 照顧計畫與 AA02 服務追蹤等資料的交換、儲存與管理。 |

### 通過條件

* 參測單位 **必須** 通過情境 1、2 其中任一情境的所有要求項目，才算通過本賽道。

#### 情境 1：AA01 照顧計畫擬定管理

AA01 照顧計畫擬定管理情境模擬個案管理師在收到照管中心轉介的長照個案後，前往案家與個案及其家屬討論，根據長照服務額度、問題清單及照顧需求擬定個人化照顧計畫的流程。在此情境中，個案管理師會透過 AA01 照顧計畫擬訂與服務連結問卷對個案進行照顧需求評估並擬定照顧計畫，後續透過標準化的資料交換機制上傳至照管中心系統。

AA01 問卷包含以下主要內容：

* 計畫目標：電聯日期、家訪日期、偕同訪視者等基本資訊
* 個案概況：身心概況、經濟收入、居住環境、社會支持等評估
* 照顧目標：擬定照顧目標與服務項目規劃
* 計畫執行規劃：長照服務核定項目、頻率及服務單位安排

本情境會使用到 QuestionnaireResponse 資源，各系統角色應具備以下核心功能：

#### 資料儲存、管理功能（LTC_REPOSITORY 角色）

* 能夠接收來自個案管理師的 AA01 照顧計畫問卷回應，包括但不限於計畫目標、個案概況、照顧目標、服務項目、頻率、服務單位等資訊。
* 支援即時或批次上傳／查詢模式，確保資料傳輸的穩定性與完整性。 
* 必須支援以 POST 方式上傳資料。
* 必須支援以 GET 方式查詢資料。
* 必須支援以 PUT 方式更新／上傳資料。
* 選擇性支援以 POST Bundle 方式上傳資料。
 
* 遵循 FHIR R4 QuestionnaireResponse 資源格式，將 AA01 照顧計畫問卷回應儲存至 FHIR Server 中，並提供查詢功能。

#### 資料上傳功能（LTC_MANAGEMENT 角色）

* 必須能以 POST 方式上傳單筆或批次 AA01 照顧計畫問卷回應。
* 必須提供介面，讓個案管理師填寫 AA01 問卷後進行資料上傳，並提供一個可驗證的介面，讓使用者確認上傳的資訊。

#### 資料查詢功能（LTC_CONSUMER 角色）

* 必須能以 GET 方式調閱 AA01 照顧計畫問卷回應。
* 必須提供查詢介面，支援以多種條件調閱 AA01 問卷回應，並支援以 _history 參數調閱歷史資料。
* 必須支援以下調閱條件： 
* 依據個案編號查詢
* 依據問卷填寫時間查詢
 

#### 情境 2：AA02 服務紀錄與追蹤管理

AA02 服務紀錄與追蹤管理情境模擬個案管理師定期追蹤長照個案的服務使用狀況與需求變化，執行服務品質追蹤，並處理相關的諮詢與申訴的流程。在此情境中，個案管理師會透過 AA02 照顧管理問卷進行電訪或訪視追蹤個案的服務適應情形，並將追蹤內容與結果記錄於系統中。

AA02 問卷包含以下主要內容：

* 追蹤服務適應與介入情形：聯繫時間、電訪對象等基本追蹤資訊
* 核定服務類別及頻率：居家服務、交通服務、專業服務、喘息服務、輔具服務等各項服務的適應情形
* 各項服務目標及整體計畫目標達成情形：長照服務使用情形追蹤與服務狀態評估
* 計畫適切性與需求異動：整體計畫的適切性評估與需求變化追蹤
* 其他處理事項：服務諮詢、申訴處理等相關事務

本情境會使用到 QuestionnaireResponse 資源，各系統角色應具備以下核心功能：

#### 資料儲存、管理功能（LTC_REPOSITORY 角色）

* 能夠接收來自個案管理師的 AA02 服務追蹤問卷回應，包括但不限於服務適應情形、服務使用狀況、計畫適切性評估、需求變化追蹤等資訊。
* 支援即時或批次上傳／查詢模式，確保資料傳輸的穩定性與完整性。 
* 必須支援以 POST 方式上傳資料。
* 必須支援以 GET 方式查詢資料。
* 必須支援以 PUT 方式更新／上傳資料。

* 遵循 FHIR R4 QuestionnaireResponse 資源格式，將 AA02 服務追蹤問卷回應儲存至 FHIR Server 中，並提供查詢功能。
